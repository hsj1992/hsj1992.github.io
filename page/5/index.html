《 JavaScript 高级程序设计》备忘-第十七章-1

错误处理与调试

JavaScript 本身是动态语言，(动态语言，是指程序在运行时可以改变其结构：新的函数可以被引进，已有的函数可以被删除等在结构上的变化。)
原本出错时给的信息都让人摸不着头脑。ECMAScript第三版引入了try-catch 和throw 语句以及一些错误类型，意在让开发人员能够适当地处理错误。
之后出现了不少JavaScript调试程序和工具。

浏览器报告的错误

主流浏览器都具有某种向用户报告 JavaScript 错误的机制。默认情况下隐藏。

IE

IE 是唯一一个在浏览器的界面窗体（chrome。chrome的意思是界面窗体?）中显示JavaScript 错误信息的浏览器。
在发生JavaScript错误时，浏览器左下角会出现一个黄色的图标，图标旁边则显示着”Error on page”（页面中有错误）。
可以双击查看相关信息（但不详细）。

对于Web开发还可以通过设置让错误对话框直接显示出来。

IE8和IE9自带脚本调试器。（更早版本需要安装）
在IE7 及更早版本中，如果错误发生在位于外部文件的脚本中，行号通常会与错误所在的行号差1。如果是嵌入在页面中的脚本发生错误，则行号就是错误所在的行号。

Firefox

默认情况下，Firefox不会通过浏览器界面给出提示。可以从后台的错误控制台看到错误。
Firefox 插件Firebug现在已经成为开发人员必备的JavaScript 纠错工具。

HTML5 脚本编程

媒体元素

Flash毒瘤，不得不用。而HTML5 新增了<audio>和<video>，为了摆脱毒瘤。
它们都提供了用于实现常用功能的JavaScript API，允许为媒体创建自定义的控件。
<video src="conference.mpg" id="myVideo">Video player not available.</video>

必需:src 属性，指向要加载的媒体文件。
可选:width和height 属性以指定视频播放器的大小，
可选:poster 属性指定图像的URI 可以在加载视频内容期间显示一幅图像。
可选:controls 属性，则意味着浏览器应该显示UI 控件，以便用户直接操作媒体。
标签之间的任何内容都将作为后备内容，在浏览器不支持这两个媒体元素的情况下显示。

因为并非所有浏览器都支持所有媒体格式，所以可以指定多个不同的媒体来源。利用<source>标签。

跨文档消息传递

跨文档消息传送（cross-document messaging），有时候简称为XDM，指的是在来自不同域的页面间传递消息。
例如，www.wrox.com域中的页面与位于一个内嵌框架中的p2p.wrox.com 域中的页面通信。
在XDM 机制出现之前，要稳妥地实现这种通信需要花很多工夫。XDM 把这种机制规范化，让我们能既稳妥又简单地实现跨文档通信。

XDM 的核心是postMessage()方法。
在HTML5 规范中，除了XDM部分之外的其他部分也会提到这个方法名，但都是为了同一个目的：向另一个地方传递数据。对于XDM而言，“另一个地方”指的是包含在当前页面中的<iframe>元素，或者由当前页面弹出的窗口。
postMessage()方法接收两个参数：一条消息和一个表示消息接收方来自哪个域的字符串。
第二个参数对保障安全通信非常重要，可以防止浏览器把消息发送到不安全的地方。

最后一行代码尝试向内嵌框架中发送一条消息，并指定框架中的文档必须来源于”http://www.wrox.com"域。
如果来源匹配，消息会传递到内嵌框架中；否则，postMessage()什么也不做。
这一限制可以避免窗口中的位置在你不知情的情况下发生改变。
